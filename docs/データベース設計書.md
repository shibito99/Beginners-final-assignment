# データベース設計書

## 1. 概要

| 項目 | 内容 |
|------|------|
| データストア | Amazon DynamoDB |
| 課金モード | PAY_PER_REQUEST（オンデマンド） |
| リージョン | ap-northeast-1（東京） |
| 管理 | Terraform `infra/modules/dynamodb/` |

DynamoDB はサーバレス構成に合わせて採用。スキーマレスで柔軟な属性追加が可能。  
リレーショナルな正規化は行わず、アクセスパターン単位でアイテムを設計（埋め込みドキュメント方式）。

### 3.1 recipe-app-recipes

#### キー設計

| 属性 | 型 | 役割 |
|------|----|------|
| `recipeId` | String (S) | パーティションキー（PK）。UUID v4 |

#### グローバルセカンダリインデックス（GSI）

| GSI名 | パーティションキー | ソートキー | 用途 |
|-------|-------------------|-----------|------|
| `genre-createdAt-index` | genre (S) | createdAt (S) | ジャンル別・新着順クエリ |
| `createdAt-index` | createdAt (S) | - | 全件新着順クエリ |

> 現在の実装では Scan + Lambda フィルタリングを使用しているため、GSI は拡張用として定義済み。

#### 属性定義

| 属性名 | 型 | 必須 | 説明 |
|-------|----|------|------|
| recipeId | String | ○ | UUID v4。PK |
| title | String | ○ | レシピタイトル |
| description | String | × | 説明文 |
| genre | String | ○ | ジャンル: japanese / western / chinese / ethnic / other |
| servings | Number | ○ | 人数（整数） |
| cookingTime | Number | × | 調理時間（分、整数） |
| tags | List<String> | × | タグ名の配列（例: `["簡単", "時短"]`） |
| ingredients | List<Map> | × | 食材リスト（下記参照） |
| steps | List<Map> | × | 調理手順リスト（下記参照） |
| imageKey | String | × | S3オブジェクトキー（例: `recipes/uuid/photo.jpg`） |
| nutrition | Map | × | 栄養情報（下記参照） |
| createdAt | String | ○ | ISO 8601 UTC（例: `2026-07-10T01:58:47Z`） |
| updatedAt | String | ○ | ISO 8601 UTC |

#### 埋め込みドキュメント

**ingredients（食材）アイテム構造**

```json
{
  "name": "鶏もも肉",
  "amount": "200",
  "unit": "g"
}
```

| 属性 | 型 | 必須 |
|------|----|------|
| name | String | ○ |
| amount | String | × |
| unit | String | × |

**steps（手順）アイテム構造**

```json
{
  "body": "鶏肉と玉ねぎを出汁で煮る"
}
```

| 属性 | 型 | 必須 |
|------|----|------|
| body | String | ○ |

**nutrition（栄養情報）マップ構造**

```json
{
  "calories": 450,
  "protein": 25,
  "fat": 12,
  "carbs": 55,
  "fiber": 3,
  "salt": 1.5
}
```

| 属性 | 型 | 単位 |
|------|----|------|
| calories | Number | kcal |
| protein | Number | g |
| fat | Number | g |
| carbs | Number | g |
| fiber | Number | g |
| salt | Number | g |

### 3.2 recipe-app-shopping-lists

#### キー設計

| 属性 | 型 | 役割 |
|------|----|------|
| `listId` | String (S) | パーティションキー（PK）。UUID v4 |

#### 属性定義

| 属性名 | 型 | 必須 | 説明 |
|-------|----|------|------|
| listId | String | ○ | UUID v4。PK |
| name | String | ○ | リスト名（例: "今週の買い物"） |
| items | List<Map> | ○ | 買い物アイテムリスト（下記参照） |
| createdAt | String | ○ | ISO 8601 UTC |
| updatedAt | String | ○ | ISO 8601 UTC |

#### 埋め込みドキュメント

**items アイテム構造**

```json
{
  "name": "鶏肉",
  "checked": false
}
```

| 属性 | 型 | 必須 | 説明 |
|------|----|------|------|
| name | String | ○ | 食材名 |
| checked | Boolean | ○ | 購入済みフラグ |

> アイテムの追加・削除・チェック状態変更はすべて `items` 配列全体を差し替える形（PUT）で行う。個別アイテムのIDは持たない。

#### アイテム例

```json
{
  "listId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "今週の買い物",
  "items": [
    { "name": "鶏肉", "checked": true },
    { "name": "卵", "checked": false },
    { "name": "玉ねぎ", "checked": false },
    { "name": "だし醤油", "checked": false }
  ],
  "createdAt": "2026-07-10T05:00:00Z",
  "updatedAt": "2026-07-10T06:15:00Z"
}
```

## 4. アクセスパターン

### レシピテーブル

| 操作 | DynamoDB 操作 | 条件 |
|------|--------------|------|
| レシピ一覧（全件） | Scan | Lambda でソート・フィルタ |
| ジャンル絞り込み | Scan + Filter | Lambda でフィルタ |
| キーワード検索 | Scan | Lambda でタイトル/説明/タグ/食材名を部分一致 |
| 調理時間フィルター | Scan + Filter | Lambda で cookingTime を比較 |
| レシピ詳細取得 | GetItem | recipeId でPK検索 |
| レシピ作成 | PutItem | recipeId = SecureRandom.uuid |
| レシピ更新 | PutItem | 既存アイテム丸ごと上書き |
| レシピ削除 | DeleteItem | recipeId でPK指定 |

### 買い物リストテーブル

| 操作 | DynamoDB 操作 | 条件 |
|------|--------------|------|
| リスト一覧（全件） | Scan | Lambda でソート |
| リスト作成 | PutItem | listId = SecureRandom.uuid |
| リスト更新（items含む） | PutItem | 丸ごと上書き |
| リスト削除 | DeleteItem | listId でPK指定 |

## 5. 注意事項

### BigDecimal 問題

DynamoDB の Ruby SDK はNumber型を `BigDecimal` として返す。  
`BigDecimal.to_json` はデフォルトで科学記法（例: `"0.2e1"`）となるため、Lambda の `normalize()` メソッドで整数または浮動小数点数に変換してからJSONシリアライズする。

```ruby
def normalize(obj)
  case obj
  when BigDecimal
    obj == obj.to_i ? obj.to_i : obj.to_f
  when Hash
    obj.transform_values { |v| normalize(v) }
  when Array
    obj.map { |v| normalize(v) }
  else
    obj
  end
end
```
