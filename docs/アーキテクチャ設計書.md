# システムアーキテクチャ設計書

## 1. 全体構成図

```
ユーザー（ブラウザ）
  │ HTTPS
  ▼
Amazon CloudFront  (d2b2m401smddic.cloudfront.net)
  │
  ├── パス: /*          → S3（フロントエンド静的ファイル）
  │     Next.js 静的エクスポート（HTML/JS/CSS）
  │
  ├── パス: /api/*      → Amazon API Gateway (HTTP API v2)
  │                          │
  │                          ▼
  │                    AWS Lambda (Ruby 3.2)
  │                    backend/lambda/handler.rb
  │                          │
  │                          ├── Amazon DynamoDB
  │                          │   recipe-app-recipes
  │                          │   recipe-app-shopping-lists
  │                          │
  │                          └── Amazon S3（画像バケット）
  │                              Presigned URL 発行のみ
  │
  └── パス: /images/*   → S3（レシピ画像）
        OAC（Origin Access Control）で保護
        ブラウザは Presigned URL で直接 PUT アップロード
```

---

## 2. アーキテクチャ選定理由

### サーバレス構成を選んだ理由

| 観点 | 内容 |
|------|------|
| コスト | Lambda・DynamoDB・API Gatewayはいずれもリクエスト課金。個人利用のアクセス量ではほぼ $0/月 |
| 運用 | EC2のOSパッチ・Rails起動管理・RDS維持が不要。インフラ管理コストがゼロに近い |
| スケーラビリティ | Lambdaは同時実行数に応じて自動スケール |
| シンプルさ | VPC・サブネット・セキュリティグループが不要。構成が最小限 |

## 3. コンポーネント詳細

### フロントエンド

| 項目 | 内容 |
|------|------|
| フレームワーク | Next.js 16.2.7（App Router） |
| ビルド方式 | 静的エクスポート（`output: "export"`） |
| ホスティング | Amazon S3 + CloudFront |
| ルーティング | CloudFront Functions でSPAルーティング（`/path` → `/path/index.html`） |

### バックエンド API

| 項目 | 内容 |
|------|------|
| 実装 | AWS Lambda（Ruby 3.2） |
| エントリーポイント | `backend/lambda/handler.rb` の `lambda_handler` 関数 |
| ルーティング | Lambda 内で `if/elsif` チェーンによりパスとHTTPメソッドを判定 |
| デプロイ | `Compress-Archive` で zip 化 → `aws lambda update-function-code` |
| 環境変数 | `RECIPES_TABLE`, `SHOPPING_LISTS_TABLE`, `IMAGES_BUCKET` |

### API Gateway

| 項目 | 内容 |
|------|------|
| タイプ | HTTP API v2 |
| 統合 | Lambda プロキシ統合（AWS_PROXY） |
| CORS | Lambda レスポンスヘッダーで `Access-Control-Allow-Origin: *` を設定 |

### データストア

| 項目 | 内容 |
|------|------|
| サービス | Amazon DynamoDB |
| テーブル | `recipe-app-recipes`, `recipe-app-shopping-lists` |
| 課金 | PAY_PER_REQUEST（使用分のみ課金） |

### 画像ストレージ

| 項目 | 内容 |
|------|------|
| サービス | Amazon S3（プライベートバケット） |
| アップロード | Lambda が Presigned PUT URL（有効期限300秒）を発行、ブラウザが直接PUT |
| 配信 | CloudFront + OAC 経由（`/images/*`） |
| バケット名 | `recipe-app-images-237228997080` |

### CDN

| 項目 | 内容 |
|------|------|
| サービス | Amazon CloudFront |
| オリジン | S3（静的ファイル）/ API Gateway（API）/ S3（画像） |
| オリジンリクエストポリシー | API Gateway向けは `Managed-AllViewerExceptHostHeader`（Hostヘッダーを除外） |
| HTTPS | CloudFrontのデフォルト証明書を使用 |

## 4. リクエストフロー

### レシピ一覧表示

```
1. ブラウザ → CloudFront (GET /)
2. CloudFront → S3（index.html を返却）
3. ブラウザでNext.jsアプリが起動
4. ブラウザ → CloudFront (GET /api/v1/recipes)
5. CloudFront → API Gateway → Lambda
6. Lambda → DynamoDB Scan
7. Lambda → CloudFront → ブラウザ（レシピJSON）
```

### 画像付きレシピ登録

```
1. ブラウザ → POST /api/v1/upload（ファイル名・ContentType）
2. Lambda → S3 Presigned PUT URL を生成して返却
3. ブラウザ → S3に直接PUT（presigned URL使用）
4. ブラウザ → POST /api/v1/recipes（imageKey を含む）
5. Lambda → DynamoDB PutItem
```

---

## 5. セキュリティ設計

| 項目 | 対策 |
|------|------|
| S3直接アクセス防止 | パブリックアクセスブロック有効、OACでCloudFront経由のみ許可 |
| API認証 | なし（個人利用・学習目的） |
| HTTPS | CloudFront経由で強制 |
| 画像アップロード | Presigned URL（短命・用途限定）、S3へのパブリック書き込み不可 |
