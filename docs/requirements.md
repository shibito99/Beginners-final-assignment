# ソフトウェア要件定義書（SRS）
## 料理レシピ管理アプリ

---

## 1. はじめに

### 1.1 目的

本ドキュメントは、料理レシピ管理アプリに関するすべての要件を定義する。本書は設計・実装・テストの基準となり、プロジェクト全体を通じて参照される。

### 1.2 スコープ

| 項目 | 内容 |
|------|------|
| プロダクト名 | 料理レシピ管理アプリ（仮称） |
| 提供形態 | Webアプリケーション（PC・スマートフォン対応） |
| 対象ユーザー | 個人（料理を記録・管理したい方） |
| 主な目的 | 自分のレシピを一元管理し、材料・手順・栄養情報を素早く参照できる |

**スコープ内（今回対象）**
- レシピの登録・閲覧・編集・削除
- レシピの検索・絞り込み
- 画像のアップロード
- 栄養素の自動計算
- 買い物リストの管理

**スコープ外（今回対象外）**
- ユーザー認証・複数ユーザー管理
- レシピの外部公開・共有
- 外部レシピサービスとの連携

### 1.3 利用シーン

- 自宅での PC 操作時：レシピの登録・整理
- 料理中のスマートフォン操作：材料・手順の確認
- 買い物時のスマートフォン操作：買い物リストの確認・チェック

### 1.4 用語定義

| 用語 | 定義 |
|------|------|
| レシピ | 料理名・材料・手順・栄養情報をまとめたデータ単位 |
| 材料 | レシピを構成する食材の名称・分量・単位の組み合わせ |
| 手順 | 料理の調理ステップ。テキストと任意の画像で構成 |
| タグ | レシピに付与できる任意のキーワードラベル |
| 買い物リスト | レシピの材料をまとめて管理するチェックリスト |
| 食品マスタ | 栄養素計算に使用する食品と成分値のマスターデータ |

---

## 2. 全体概要

### 2.1 プロダクトの概要

料理レシピを登録・管理するWebアプリケーション。レシピは画像・材料・手順とともに保存でき、食材名やジャンルで素早く検索できる。材料から栄養素を自動計算する機能と、レシピ材料を買い物リストに連携する機能を備える。

### 2.2 ユーザー像

**プライマリユーザー**: 料理が好きで、自分のオリジナルレシピや試した料理を記録・管理したい個人。調理中にスマートフォンで手順を確認したり、スーパーで買い物リストを確認したりする使い方を想定する。

### 2.3 前提条件・制約

| 区分 | 内容 |
|------|------|
| 前提条件 | インターネット接続環境があること |
| 前提条件 | 対象ブラウザ: Chrome / Safari / Firefox の最新版 |
| 制約 | 初期フェーズはユーザー認証なし（個人利用・自己ホスティング） |
| 制約 | フロントエンドは静的エクスポート（SSR不使用）のため、SEO対応は対象外 |

### F1 — レシピ登録

**目的**: 新しいレシピを登録する。

**入力仕様**

| フィールド | 型 | 必須 | 制約 |
|-----------|-----|------|------|
| タイトル | 文字列 | ○ | 最大100文字 |
| 説明文 | テキスト | × | 最大500文字 |
| ジャンル | 選択肢 | ○ | 和食 / 洋食 / 中華 / イタリアン / その他 |
| タグ | 文字列（複数） | × | 1タグ最大20文字、最大10個 |
| 調理時間（分） | 数値 | × | 1〜999 |
| 人数（人前） | 数値 | ○ | 1〜99 |
| 材料 | 配列 | ○ | 最低1件。各行：食材名・分量・単位 |
| 手順 | 配列 | ○ | 最低1件。各行：テキスト（最大300文字）・任意画像 |
| メイン画像 | 画像ファイル | × | JPEG / PNG / WebP、最大5MB |

**正常系フロー**
1. ユーザーが登録フォームを開く
2. 各項目を入力する
3. 「登録する」ボタンを押す
4. バリデーション通過後、DBに保存
5. 登録完了メッセージを表示し、レシピ詳細画面へ遷移

**異常系・エラー処理**

| 条件 | 表示メッセージ |
|------|-------------|
| 必須項目未入力 | 「〇〇は必須です」（フォーム送信をブロック） |
| 画像サイズ超過 | 「5MB以下の画像を選択してください」 |
| 画像形式不正 | 「JPEG・PNG・WebP 形式のみ対応しています」 |
| 通信エラー | 「保存に失敗しました。もう一度お試しください」 |

---

### F2 — レシピ一覧表示

**目的**: 登録済みレシピをカード形式で一覧表示する。

**表示仕様**
- カードに表示する情報: サムネイル画像（未設定時はデフォルト画像）・タイトル・ジャンル・調理時間・登録日
- 1ページあたり12件表示（無限スクロールまたはページネーション）
- 並び替え: 新着順（デフォルト）/ タイトル昇降順 / 調理時間の短い順

**正常系フロー**
1. ユーザーがホーム画面を開く
2. 最新のレシピが一覧で表示される
3. カードをクリックするとレシピ詳細画面へ遷移

**異常系・エラー処理**

| 条件 | 対応 |
|------|------|
| 登録レシピ0件 | 「まだレシピが登録されていません。最初のレシピを登録しましょう！」を表示 |
| データ取得失敗 | 「データの取得に失敗しました」と再試行ボタンを表示 |

---

### F3 — レシピ詳細表示

**目的**: レシピの全情報を1画面で確認できる。

**表示内容**
- メイン画像
- タイトル・説明文・ジャンル・タグ・調理時間・人数
- 材料一覧（チェックボックス付き）
- 手順（番号付き、各ステップ画像があれば表示）
- 栄養情報（1人前）
- 編集・削除ボタン
- 「買い物リストに追加」ボタン

---

### F4 — レシピ編集・削除

**目的**: 登録済みレシピの内容を更新または削除する。

**編集仕様**
- F1 と同じ入力フォームに既存データを初期値として表示する
- 「更新する」ボタン押下で上書き保存する
- バリデーション仕様は F1 と同じ

**削除仕様**
- 削除ボタン押下で確認ダイアログを表示する（「本当に削除しますか？この操作は取り消せません」）
- 確認後に削除を実行し、レシピ一覧画面へ遷移する
- 関連する材料・手順・栄養情報も連動削除する（CASCADE）

**異常系・エラー処理**

| 条件 | 表示メッセージ |
|------|-------------|
| 更新失敗 | 「更新に失敗しました。もう一度お試しください」 |
| 削除失敗 | 「削除に失敗しました。もう一度お試しください」 |

---

### F5 — レシピ検索・絞り込み

**目的**: キーワードや条件でレシピをすばやく見つける。

**検索・絞り込み仕様**

| 種別 | 対象 | マッチ方式 |
|------|------|----------|
| キーワード検索 | タイトル・説明文・タグ | 部分一致 |
| 食材名検索 | 材料の食材名 | 部分一致 |
| ジャンル絞り込み | ジャンル | 完全一致 |
| タグ絞り込み | タグ | 完全一致（複数選択可） |
| 調理時間絞り込み | 調理時間 | ～15分 / ～30分 / ～60分 / 60分以上 |

**動作仕様**
- 検索バー入力は300msのデバウンスでリアルタイム検索する
- 複数条件はAND検索で処理する
- 検索結果0件の場合は「該当するレシピが見つかりませんでした」を表示する

---

### F6 — 画像アップロード

**目的**: レシピ登録・編集時に料理写真を添付する。

**仕様**
- 対応形式: JPEG / PNG / WebP
- 最大ファイルサイズ: 5MB
- アップロード後にサーバーサイドでリサイズする（サムネイル用: 300×300px、詳細用: 800px幅）
- ドラッグ＆ドロップとファイル選択ダイアログの両方に対応する
- アップロード前にプレビューを表示する

**異常系・エラー処理**

| 条件 | 表示メッセージ |
|------|-------------|
| サイズ超過 | 「5MB以下の画像を選択してください」 |
| 形式不正 | 「JPEG・PNG・WebP 形式のみ対応しています」 |
| アップロード失敗 | 「画像のアップロードに失敗しました。再度お試しください」 |

---

### F7 — 栄養素計算

**目的**: 登録材料の分量から1人前の主要栄養素を自動計算・表示する。

**表示栄養素**

| 栄養素 | 単位 |
|--------|------|
| エネルギー | kcal |
| タンパク質 | g |
| 脂質 | g |
| 炭水化物 | g |
| 食物繊維 | g |
| 食塩相当量 | g |

**計算仕様**
- 食品マスタ（文部科学省「日本食品標準成分表」準拠）をDBに保持する
- 材料名で食品マスタを検索し、分量・単位から100gあたりの成分値を按分して計算する
- マスタに存在しない材料はスキップし、「一部の材料は栄養計算の対象外です」と注記する

**表示タイミング**
- レシピ詳細画面に常時表示する
- 登録・編集フォームでリアルタイムプレビューとして表示する

---

### F8 — 買い物リスト連携

**目的**: レシピの材料を買い物リストとして管理し、買い物中の利便性を高める。

**機能仕様**

| 機能 | 仕様 |
|------|------|
| 材料一括追加 | レシピ詳細画面から「買い物リストに追加」ボタンで材料を一括追加する |
| 個別追加 | 材料ごとのチェックボックスで選択してから追加できる |
| 人数換算 | 追加時に人数を指定すると分量を自動換算する |
| チェック | 購入済みをチェックボックスでマークできる |
| 削除 | アイテム個別削除・購入済みアイテムの一括削除ができる |
| 複数リスト | 複数のリストを作成・切り替えられる（例：「今週の献立」「週末分」） |

**正常系フロー**
1. ユーザーがレシピ詳細画面で「買い物リストに追加」を押す
2. 人数入力ダイアログが表示される
3. 人数確認後、材料が換算されて買い物リストに追加される
4. 買い物リスト画面で追加された材料を確認・管理できる

## 4. 非機能要件

### 4.1 パフォーマンス

| 指標 | 目標値 | 計測方法 |
|------|--------|---------|
| レシピ一覧表示（1,000件） | ≤ 2秒 | ブラウザの DevTools Network タブ |
| 検索応答時間 | ≤ 1秒 | 同上 |
| 画像アップロード（5MB） | ≤ 10秒 | 同上 |
| 栄養素計算（材料20件） | ≤ 1秒 | サーバーログ |

### 4.2 可用性・信頼性

- 稼働率目標: 99%（個人利用のため月に7〜8時間の停止まで許容）
- 障害発生時は nginx のエラーページを表示し、ユーザーに状態を通知する
- RDS の自動バックアップ（7日間保持）によりデータ損失リスクを低減する

### 4.3 セキュリティ

- 初期フェーズは個人利用・自己ホスティングのため認証なし
- 将来の公開時に備え、認証追加を容易にする設計とする（JWT / OAuth を拡張ポイントとして設計）
- サーバーサイドで入力値をバリデーションし、SQLインジェクション・XSSを防止する
- 画像ファイルはMIMEタイプとマジックバイトの両方を検証する
- S3画像バケットはプライベート設定とし、CloudFront 経由のみ公開する
- EC2 への管理アクセスは SSM Session Manager 経由とし、SSHポートを開放しない

### 4.4 保守性・拡張性

- コンポーネント単位で責務を分割し、再利用できる設計とする
- ビジネスロジックはUI層から分離する（Rails: サービスクラス / Next.js: カスタムフック）
- 設定値は環境変数で管理し、コードにハードコードしない
- APIはバージョニング（/api/v1/）し、将来の変更時に後方互換性を維持できるようにする

### 4.5 ユーザビリティ・アクセシビリティ

- モバイルファースト設計とし、スマートフォンでの操作を優先する
- 最小タップターゲット: 44×44px（Apple HIG 準拠）
- 本文フォントサイズ: 16px 以上
- コントラスト比: WCAG AA 準拠（4.5:1 以上）
- ローディング中はスケルトンスクリーンまたはスピナーを表示する

### 4.6 運用・監視

- アプリケーションログは EC2 上のファイル（Nginx + Puma）に出力する
- AWS Budgets でコストアラートを設定し、月$50超えで通知する
- RDS 自動バックアップを有効にし、7日間保持する
