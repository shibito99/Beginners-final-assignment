# インフラ設計書（Terraform）

## 1. 概要

| 項目 | 内容 |
|------|------|
| IaCツール | Terraform >= 1.7 |
| AWSプロバイダー | hashicorp/aws ~> 5.0 |
| tfstate管理 | S3バックエンド（`recipe-app-tfstate-237228997080`） |
| リージョン | ap-northeast-1（東京） |
| ディレクトリ | `infra/` |

---

## 2. ディレクトリ構成

```
infra/
├── main.tf              # ルートモジュール（モジュール呼び出し）
├── variables.tf         # 変数定義
├── outputs.tf           # 出力値定義
├── terraform.tfvars     # 変数値（gitignore対象）
├── backend.tf           # S3バックエンド設定
└── modules/
    ├── s3/              # S3バケット（フロントエンド・画像）
    ├── cloudfront/      # CloudFront Distribution + OAC
    ├── api_gateway/     # API Gateway (HTTP API v2)
    ├── lambda/          # Lambda 関数・IAMロール・実行権限
    └── dynamodb/        # DynamoDB テーブル
```

> VPC・サブネット・セキュリティグループ・EC2・RDSは使用しない。完全サーバレス構成。

## 3. モジュール詳細

---

### 3.1 s3 モジュール

**作成リソース**

| リソース | 内容 |
|---------|------|
| `aws_s3_bucket.frontend` | Next.js 静的ファイル配信用バケット |
| `aws_s3_bucket.images` | レシピ画像保存用バケット |
| `aws_s3_bucket_public_access_block` | 両バケットともパブリックアクセスブロック有効 |
| `aws_s3_bucket_policy.frontend` | CloudFront OACからのGetObjectのみ許可 |
| `aws_s3_bucket_policy.images` | CloudFront OACからのGetObjectのみ許可 |
| `aws_s3_bucket_cors_configuration.images` | Presigned URL PUTアップロード用CORSルール |

**CORS設定（画像バケット）**

```hcl
cors_rule {
  allowed_headers = ["*"]
  allowed_methods = ["PUT", "GET", "HEAD"]
  allowed_origins = ["*"]
  max_age_seconds = 3000
}
```

### 3.2 cloudfront モジュール

**作成リソース**

| リソース | 内容 |
|---------|------|
| `aws_cloudfront_origin_access_control.s3` | OAC（S3向け） |
| `aws_cloudfront_distribution.main` | CloudFront Distribution本体 |
| `aws_cloudfront_function.spa_router` | SPA向けルーティング関数 |

**オリジン設定**

| オリジンID | オリジン | パスパターン | オリジンリクエストポリシー |
|-----------|---------|------------|--------------------------|
| `s3-frontend` | S3フロントエンドバケット | `/*`（デフォルト） | なし（OACでアクセス） |
| `api-gateway` | API Gateway エンドポイント | `/api/*` | `Managed-AllViewerExceptHostHeader` |
| `s3-images` | S3画像バケット | `/images/*` | なし（OACでアクセス） |

> `Managed-AllViewer`ではなく`Managed-AllViewerExceptHostHeader`を使用する理由:  
> CloudFrontのHostヘッダーをそのままAPI Gatewayに転送するとAPI Gatewayが`403 Forbidden`を返すため。

**CloudFront Functions（SPAルーティング）**

拡張子なし・`/api/`以外のパスを `{path}/index.html` に書き換える関数。  
Next.js静的エクスポートのディレクトリ構造に対応するためのルーティング。

### 3.4 lambda モジュール

**作成リソース**

| リソース | 内容 |
|---------|------|
| `aws_iam_role.lambda` | Lambda実行ロール |
| `aws_iam_role_policy.dynamodb` | DynamoDB読み書き権限ポリシー |
| `aws_iam_role_policy.s3_images` | 画像S3への読み書き・Presigned URL発行権限 |
| `aws_lambda_function.api` | Lambda関数本体 |
| `aws_lambda_permission.api_gateway` | API GatewayからのInvoke権限 |

**Lambda関数設定**

| 項目 | 値 |
|------|-----|
| 関数名 | `recipe-app-api` |
| ランタイム | `ruby3.2` |
| ハンドラー | `handler.lambda_handler` |
| タイムアウト | 30秒 |
| メモリ | 128 MB |

**環境変数**

| 変数名 | 値 |
|-------|-----|
| `RECIPES_TABLE` | `recipe-app-recipes` |
| `SHOPPING_LISTS_TABLE` | `recipe-app-shopping-lists` |
| `IMAGES_BUCKET` | `recipe-app-images-237228997080` |

**Lambdaコードのデプロイ方法**

Terraform の `archive` プロバイダーは使用せず、PowerShell で手動zip作成：

```powershell
Compress-Archive -Path backend/lambda/* -DestinationPath infra/modules/lambda/lambda.zip -Force
```

### 3.5 dynamodb モジュール

**作成リソース**

| リソース | テーブル名 |
|---------|-----------|
| `aws_dynamodb_table.recipes` | `recipe-app-recipes` |
| `aws_dynamodb_table.shopping_lists` | `recipe-app-shopping-lists` |

**recipe-app-recipes テーブル設定**

```hcl
billing_mode = "PAY_PER_REQUEST"
hash_key     = "recipeId"

attribute { name = "recipeId" type = "S" }
attribute { name = "genre"     type = "S" }
attribute { name = "createdAt" type = "S" }

global_secondary_index {
  name            = "genre-createdAt-index"
  hash_key        = "genre"
  range_key       = "createdAt"
  projection_type = "ALL"
}

global_secondary_index {
  name            = "createdAt-index"
  hash_key        = "createdAt"
  projection_type = "ALL"
}
```

**recipe-app-shopping-lists テーブル設定**

```hcl
billing_mode = "PAY_PER_REQUEST"
hash_key     = "listId"

attribute { name = "listId" type = "S" }
```

## 5. インフラ適用手順

```bash
cd infra/
terraform init
terraform plan
terraform apply
```

初回 `terraform apply` でCloudFront Distributionの作成に5〜10分かかる。
